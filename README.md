## Flagship Project

### Reflux | Distributed Asynchronous Event Processing Engine

Reflux is my main systems-engineering project: a distributed, asynchronous event processing engine inspired by real-world backend infrastructure patterns used in large-scale systems. It is built to demonstrate concurrency control, queue-driven execution, worker-pool orchestration, and monitoring-first design.

<p align="center">
  <img src="https://img.shields.io/badge/Architecture-Producer_%E2%86%92_Queue_%E2%86%92_Scheduler_%E2%86%92_Worker_Pool_%E2%86%92_Processing_Engine_%E2%86%92_Monitoring-0d1117?style=for-the-badge&logo=apachekafka&logoColor=white" alt="Reflux Architecture" />
</p>

**Core capabilities**

- Publisher-subscriber architecture for event ingestion and dispatch
- Asynchronous event processing with multi-threaded worker pools
- Priority task scheduling and queue-based execution paths
- Dynamic processing pipelines for extensible event handling
- Retry mechanisms and concurrent task management
- WebSocket-backed monitoring dashboard for runtime visibility

**Technology stack**

- Java
- Spring Boot
- ExecutorService
- BlockingQueue
- CompletableFuture
- Docker
- WebSockets

**Why this project matters**

Reflux is designed as infrastructure, not just an application. The emphasis is on throughput, coordination, fault tolerance, observability, and clean separation of concerns across the event pipeline.
